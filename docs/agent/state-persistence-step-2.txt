Step 2: Redis-backed persistence with TTL

What was added:
- Added createRedisConversationStore() in apps/agent/server/conversation-store.ts.
- Added createConversationStoreFromEnv() factory with memory fallback.
- Added Redis key layout:
  - agent:conversation:messages:{conversationId}
  - agent:conversation:state:{conversationId}
- Added TTL support (default 24h, configurable).

Runtime config:
- AGENT_CONVERSATION_STORE=redis|memory
- AGENT_REDIS_URL (fallbacks to REDIS_URL)
- AGENT_CONVERSATION_TTL_MS

Index wiring:
- apps/agent/server/index.ts now builds store from env and injects into createAgent().

Why:
- Conversation and workflow state now survive process restarts.
- Supports horizontal scaling where multiple agent instances share Redis state.
